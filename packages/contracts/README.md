# Пакет контрактов

Этот пакет содержит общие API-контракты, которые используются и фронтендом, и бэкендом.

## Экспорты
1. `@skillswap/contracts/auth` — общие auth-типы (payload запросов и ответов).
2. `@skillswap/contracts/authSchemas` — общие runtime-схемы Zod для валидации auth-запросов.
3. `@skillswap/contracts/materials` — общие типы методических материалов.
4. `@skillswap/contracts/materialSchemas` — общие runtime-схемы Zod для валидации запросов материалов.
5. `@skillswap/contracts/ratings` — общие типы оценок завершённых обменов.
6. `@skillswap/contracts/ratingSchemas` — общие runtime-схемы Zod для валидации запросов оценок.

## Источник истины
1. Контракты транспортного уровня для auth хранятся здесь.
2. Фронтенд и бэкенд должны импортировать типы из этого пакета, а не переопределять их локально.
3. Бэкенд должен использовать экспортируемые здесь схемы для валидации входящих данных.
4. Контракты материалов используют типы `theory`, `practice`, `testing`; новые типы материала добавляются только через этот пакет.
5. Контракты рейтингов не принимают `ratedUserId` от клиента: оцениваемый пользователь вычисляется бэкендом по участникам обмена.

## Правила изменений
1. Любое изменение контракта — сквозное изменение между слоями.
2. В одном PR обновляйте парсинг и валидацию запросов на бэкенде.
3. В одном PR обновляйте backend-тесты.
4. В одном PR обновляйте API-типизацию фронтенда и места использования.
5. Сначала предпочитайте обратносовместимые (additive) изменения.
6. Для breaking changes добавляйте заметку о миграции в описании PR.

## Проверка
Запускать из корня репозитория:

1. `npm run backend:test`
2. `npm run backend:build`
3. `npm run test -- --runInBand`
4. `npm run build`
